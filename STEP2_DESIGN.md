# Step2 设计：从聚类到 Workflow 提取

## 目标

将 Step1 的聚类结果转换为 Workflower skill 可处理的输入目录，并支持批量任务提交。

## 输入

- **cluster_metadata** Lance 表：包含所有聚类的元数据
- **chain_embeddings** Lance 表：包含所有思维链的向量和元数据
- **TOS 存储**：`paper_ocr/xml/` 和 `paper_ocr/md/` 中的原始文件

## 输出

为每个聚类生成一个工作目录，结构如下：

```
data/{domain}/workflows/cluster_{local_cluster_id}/
├── selected_chains.json       # 聚类中的所有链
├── md/                         # 论文 Markdown 文件
│   ├── {paper_id_1}.md
│   ├── {paper_id_2}.md
│   └── ...
└── xml/                        # 思维链 XML 文件
    ├── {paper_id_1}_{conclusion_id}.xml
    ├── {paper_id_2}_{conclusion_id}.xml
    └── ...
```

## selected_chains.json 格式

```json
[
  {
    "chain_id": "812454164008271872_c1",
    "paper_id": "812454164008271872",
    "cluster_id": 0,
    "journal": "Nature Methods",
    "conclusion_id": "c1",
    "conclusion_title": "qRT-PCR validation of microarray results",
    "chain_text": "Step 1: Select candidate genes...\nStep 2: Design primers...",
    "num_steps": 7,
    "has_citations": true,
    "has_figures": false
  }
]
```



## 任务提交方式

### 方式 1: 顺序执行脚本

生成 `run_workflows.sh`:

```bash
#!/bin/bash
# Auto-generated by Step2

cd data/bioinformatics/workflows/cluster_0
echo "Processing cluster_0..."
# 这里需要用户手动调用 Workflower skill
# 或者通过 Claude Code API（如果有）

cd ../cluster_1
echo "Processing cluster_1..."
# ...
```

生成 `workflow_tasks.json`:

用户可以：
1. 手动逐个进入目录，调用 Workflower
2. 编写自动化脚本（如果 Claude Code 支持 API 调用）
3. 使用任务调度系统（如 Airflow）

## 关键设计决策

### 1. 聚类选择策略

- **默认**: 处理所有聚类
- **可选过滤**: 按大小、质量指标、手动指定 ID

### 2. 文件下载策略

- **增量下载**: 检查本地缓存，避免重复下载
- **并发下载**: 使用 asyncio + httpx 并发下载
- **错误处理**: 下载失败的文件记录到日志，不阻塞整体流程

### 3. selected_chains.json 格式

- 包含完整元数据（journal, conclusion_id, num_steps 等）
- 与 Workflower 期望的格式兼容
- 保留 cluster_id 用于追溯

### 4. 任务提交方式

- **不自动调用 Workflower**: 因为 Workflower 是交互式 skill，需要用户确认
- **生成任务清单**: 用户可以手动或通过脚本逐个处理
- **未来扩展**: 如果 Claude Code 提供 API，可以实现自动批量提交

## 与 Step1 的衔接

Step1 输出：
- `cluster_metadata` Lance 表
- `chain_embeddings` Lance 表（带 cluster_id）

Step2 输入：
- 读取 `cluster_metadata` 获取聚类列表
- 读取 `chain_embeddings` 获取每个聚类的链

## 与 Workflower 的衔接

Step2 输出：
- 符合 Workflower 期望的目录结构
- `selected_chains.json` 包含所有必需字段
- `md/` 和 `xml/` 文件已下载到本地

用户操作：
1. 进入 `data/{domain}/workflows/cluster_{id}/`
2. 调用 Workflower skill（通过 Claude Code）
3. Workflower 自动检测输入文件并执行 3 个阶段

## 实现优先级

1. **Phase 1**: 基础功能
   - cluster_loader.py
   - workflow_dir_builder.py（不含 TOS 下载，假设文件已存在）
   - pipeline.py（基础流程）

2. **Phase 2**: 文件下载
   - 集成 TOS 客户端
   - 实现增量下载和缓存

3. **Phase 3**: 任务管理
   - task_generator.py
   - 生成任务清单和脚本

4. **Phase 4**: 高级功能
   - 聚类质量过滤
   - 任务状态追踪
   - 失败重试机制
