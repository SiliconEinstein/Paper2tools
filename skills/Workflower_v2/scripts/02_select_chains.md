# 阶段二：筛选核心推理链

**目标**：确定哪些具体推理链和论文构成核心工作流。

**前置条件**：阶段一返回"可提取"

---

## 输入

- `selected_chains.json`
- 阶段一输出（workflow_summary、support_count）

---

## 操作步骤

### 第一步：筛选支撑链

从所有推理链中，选取满足以下条件的链：
- 遵循阶段一识别的核心工作流模式
- 包含足够细节（而非仅高层提及）
- 内部一致（无矛盾步骤）

**目标**：选取 ≥10 条链（最低支撑阈值）

### 第二步：提取论文 ID

对每条选中的链，记录：
- `paper_id`（来自链的元数据）
- `conclusion_id`（如有）
- 该链的贡献简述

### 第三步：论文去重

若多条链来自同一篇论文：
- 保留最详细的那条链
- 标注该论文对工作流有强支撑

### 第四步：梳理公共步骤

跨所有选中链，识别：
- 哪些步骤出现在**所有**链中（核心步骤）
- 哪些步骤出现在**部分**链中（可选/变体步骤）
- 规范的步骤顺序是什么？

**输出**：初步步骤序列

---

## 输出格式

```json
{
  "selected_chains": [
    {
      "chain_id": "chain_0",
      "paper_id": "{论文ID}",
      "conclusion_id": "{结论ID}",
      "contribution": "{该链的贡献描述}"
    }
  ],
  "unique_papers": [
    {
      "paper_id": "{论文ID}",
      "chain_count": 2,
      "note": "{论文简要说明}"
    }
  ],
  "preliminary_steps": [
    {
      "step_id": 1,
      "name": "{步骤名称}",
      "appears_in": "12/15 条链",
      "core": true
    },
    {
      "step_id": 5,
      "name": "{步骤名称}",
      "appears_in": "8/15 条链",
      "core": false
    }
  ]
}
```

---

## 链质量判断标准

**纳入条件**：
- 提及具体方法/工具（而非"我们进行了测量"）
- 体现步骤依赖关系（"利用磁场数据，我们进一步..."）
- 包含定量细节（磁场范围、温度等）

**排除条件**：
- 过于模糊（"我们进行了标准分析"）
- 与核心模式矛盾
- 不完整（缺少关键步骤）

---

## 进入阶段三的验证

- [ ] 已选取 ≥10 条链
- [ ] 链来自 ≥5 篇不同论文（多样性检查）
- [ ] 初步步骤构成连贯序列
- [ ] 每个核心步骤在 ≥80% 的选中链中出现
